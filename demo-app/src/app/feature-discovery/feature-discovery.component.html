Tap target

Feature discovery prompts have more impact when they are presented to the right users at contextually relevant moments. When presented to the wrong user at the wrong time, they can be intrusive and annoying.

I am here

Provide value and encourage return visits by introducing users to new features and functionality at contextually relevant moments.
HTML Structure

Feature discovery visibility is triggered progammatically using the open and close methods on the mz-feature-discovery component by using a template reference variable as shown on the example below. Specify the element to emphase by providing its id through the targetId property of mz-feature-discovery component.

<!-- open/close buttons --> <button mz-button (click)="featureDiscovery.open()">Open tap target</button> <button mz-button (click)="featureDiscovery.close()">Close tap target</button> <!-- tap target --> <button id="star-button" mz-button [float]="true" [large]="true"> <i mz-icon-mdi [icon]="'star'"></i> </button> <!-- mz-feature-discovery component --> <mz-feature-discovery [targetId]="'star-button'" #featureDiscovery> <h5>I am here</h5> <p>Provide value and encourage return visits by introducing users to new features and functionality at contextually relevant moments.</p> </mz-feature-discovery>

Methods

The mz-feature-discovery component expose methods to programmatically interact with feacture discovery. To use them you need to add a template reference variable on the mz-feature-discovery element. Refer to the HTML Structure section above for a concrete example.

open(): void

Opens feature discovery

close(): void

Closes feature discovery
