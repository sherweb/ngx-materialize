Input

Ng2-Materialize leverage the use of the native HTML input by using the mz-input directive directly on the input element tag to ease the fact that you might use third party library that would ask you to add properties directly on that native input element.

Basic input

Label and placeholder can be combined using both label and placeholder properties.

To use the floating label directly as a placeholder provide only the label property.

You can omit the floating label and use only a placeholder by providing only the placeholder property.

Inline input
Input can be inline:

To show the input inline set inline property to true on mz-input-container component.

Disabled input

Disable the input by using the native property disabled.

HTML5 input type

All the HTML5 input types are supported with the native property type as shown with this example using type password.

Validate

Set validate property to true to leverage HTML5 type validation and change the border color depending on the validity state.

Error/Success Messages

You can provide Error/Success hints using dataError and/or dataSuccess properties with the desired text.

Icon Prefixes

To use an icon prefix add the icon inside the mz-input-container component with the mz-input-prefix directive on the icon tag.

Character Count

You can add a character count on input focus using the native property length with the number of restricted characters.

Autocomplete

Add an autocomplete dropdown bellow the input to suggest possible values by providing the autocomplete property with the proper object structure (see input properties table).

HTML Structure

The input element must be contained inside the mz-input-container component otherwise an error will be shown in the console.

<mz-input-container class="col s12"> <i mz-icon-mdi mz-input-prefix [icon]="'mail'"> <i> <input mz-input [label]="'Email'" [validate]="true" [dataError]="'Email is invalid'" [dataSuccess]="'Email is valid'" id="email-input" placeholder="Enter your email" length="150" type="email"> </mz-input-container>
Properties
mz-input-container

The mz-input-container component can be customized using the following properties.

Property Mandatory Type Description Default value
inline boolean Show input inline false
mz-input

Inputs can be customized using the following properties on the input element when adding mz-input directive.

Property Mandatory Type Description Default value
autocomplete { data: { [key: string]: string }, limit: number } Suggest possible values
dataError string Invalid message
dataSuccess string Valid message
id string Id of the input
label string Floating label
length number Show character count
placeholder string Show placeholder
type string HTML5 input type
validate boolean Activate HTML5 validation false
