Media

Ng2-materialize provide the MzMediaService to evaluate whether a given media (small, medium, large, extra-large) is true or false according to current device screen resolution.

Demo

Below you will find the value of the media observable property available with the MzMediaService. Resize your browser to see the value changes according to your view port.

{{ media | async | json }}

Using isActive(breakpoing: string) method in MzMediaService you can evaluate if a given breakpoint is true or false according to current device's resolution. Resize your browser to see the true/false value change according to the screen resolution.

mediaService.isActive('gt-s') => {{ greaterThanSmall | async }}
mediaService.isActive('gt-m') => {{ greaterThanMedium | async }}
mediaService.isActive('gt-l') => {{ greaterThanLarge | async }}
MzMediaService
isActive(breakpoint: string): Observable<boolean>
Parameter Type Description
breakpoint string The media breakpoint to evaluate
Returns Description
Observable<boolean> A boolean observable representing whether or not the given breakpoint is true of false
media: Observable<Media>
Returns Description
Observable<Media> An observable object containing current window height, width and media size (s, l, m or xl)
Usage
example.component.ts
public smallResolution: Observable<boolean>; constructor(private mediaService: MzMediaService) { this.smallResolution = this.mediaService.isActive('s'); // small screen resolution }
example.component.html
<div *ngIf="smallResolution | async"> I am only visible on a small resolution device </div>
Media Breakpoints

Breakpoints use the same screen width as Materialize Screen Sizes

Breakpoint Screen Size
s or lt-m >= 0px and <= 600px
gt-s > 600px
m > 600px and <= 992px
gt-m > 992px
lt-l >= 0px and <= 992px
l > 992px and <= 1200px
gt-l or xl > 1200px
lt-xl >= 0px and <= 1200px
