You can set the modal to open in fullscreen by setting the fullscreen input property to true on the mz-modal element. When used with basic or fixed footer modals the aspect of modal will be kept with a padding around the modal while when applied on bottom sheet modals it won't keep any padding and behave like a page transition. Note that you can make it fullscreen only on specific screen resolution by using MzMediaService.

To customize the behavior of each modal you can use the options input property and provide a Materialize.ModalOptions object on the mz-modal element.

public modalOptions: Materialize.ModalOptions = { dismissible: false, // Modal can be dismissed by clicking outside of the modal opacity: .5, // Opacity of modal background inDuration: 300, // Transition in duration outDuration: 200, // Transition out duration startingTop: '100%', // Starting top style attribute endingTop: '10%', // Ending top style attribute ready: (modal, trigger) => { // Callback for Modal open. Modal and trigger parameters available. alert('Ready'); console.log(modal, trigger); }, complete: () => { alert('Closed'); } // Callback for Modal close };
Modal Close Directive

To make an action button or anchor to close the modal, just add the mz-modal-close directive to your button or anchor.

Properties

Modals can be customized using the following properties on the mz-modal element.

Methods

The mz-modal component expose methods to programmatically interact with the modal. To use them you need to add a template reference variable on the mz-modal element. Refer to the HTML Structure section above for a concrete example.

openModal(): void

Opens the modal

closeModal(): void

Closes the modal

MzModalService

Alternatively, the service MzModalService allow you to trigger modals programmatically without having to add the modal directly to the DOM of your page. When closing the modal the service will automatically take care of removing it from the DOM. To do so you will need to create a component that extends MzBaseModal and pass it to the open method of the service. Aslo, as the component will be created dynamically, don't forget to add the component to load as an entryComponents in your module to allow template compiler to include it in the compilation.

Inspect the DOM to see that app-modal-example component is added inside the body element when modal is opened and then removed when closed.

example.component.html

<button mz-button (click)="openServiceModal()">Modal Service</button>

example.component.ts

import { Component } from '@angular/core'; import { MzModalService } from 'ngx-materialize'; import { ModalExampleComponent } from './modal-example/modal-example.component'; @Component({ ... }) export class ExampleComponent { constructor(private modalService: MzModalService) { } public openServiceModal() { this.modalService.open(ModalExampleComponent); } }
