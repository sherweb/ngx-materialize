Datepicker

import { MzDatepickerModule } from 'ngx-materialize'

Ngx-Materialize leverage the use of the native HTML input by using the mz-datepicker directive directly on the input element tag to ease the fact that you might use third party library that would ask you to add properties directly on that native input element.

Basic datepicker

Label and placeholder can be combined using both label and placeholder properties.

To use the floating label directly as a placeholder provide only the label property.

You can omit the floating label and use only a placeholder by providing only the placeholder property.

Inline datepicker
Datepicker can be inline:

To show the input inline set inline property to true on mz-datepicker-container component.

Disabled datepicker

Disable the datepicker input by using the native property disabled.

Icon Prefixes

To use an icon prefix add the icon inside the mz-datepicker-container component with the mz-input-prefix directive on the icon tag.

Datepicker options

To customize the behavior of a datepicker you can use the options input property and provide a Pickadate.Options object with the mz-datepicker directive.

Materialize uses pickadate.js library to create the materialized date picker. You can find a complete list of available options with how to use it by following this link: http://amsul.ca/pickadate.js/date/#options.
public datepickerOptions: Pickadate.DateOptions = { clear: 'Clear', // Clear button text close: 'Ok', // Ok button text today: 'Today', // Today button text closeOnClear: true, closeOnSelect: false, format: 'dddd, dd mmm, yyyy', // Visible date format (defaulted to formatSubmit if provided otherwise 'd mmmm, yyyy') formatSubmit: 'yyyy-mm-dd', // Return value format (used to set/get value) onClose: () => alert('Close has been invoked.'), onOpen: () => alert('Open has been invoked.'), selectMonths: true, // Creates a dropdown to control month selectYears: 10, // Creates a dropdown of 10 years to control year, };
Working with NgControl
Set datepicker value

To assign a value to the datepicker, you will need to use same format specified in the formatSubmit property of the options input binding. If no formatSubmit is provided it will need to use the format option property or its default value.

example.component.html

<mz-datepicker-container class="col s12"> <input mz-datepicker [label]="'Date of birth'" [options]="options" [(ngModel)]="dateOfBirth" id="datepicker" placeholder="Select your birth date" type="text"> </mz-datepicker-container>

example.component.ts

public options: Pickadate.DateOptions = { format: 'dddd, dd mmm, yyyy', formatSubmit: 'yyyy-mm-dd', }; public dateOfBirth = '2017-08-12'; // use formatSubmit format to set datepicker value
Get datepicker value

The returned value of the selected date will be formatted as specified with the formatSubmit property of the options input binding. If no formatSubmit is provided it will use the format option property or its default value.

HTML Structure

The input element must be contained inside the mz-datepicker-container component otherwise an error will be shown in the console.

<mz-datepicker-container class="col s12"> <i mz-icon-mdi mz-input-prefix [icon]="'calendar'"> </i> <input mz-datepicker [label]="'Date of birth'" [options]="options" id="datepicker" placeholder="Select your birth date" type="text"> </mz-datepicker-container>
